About Me

Hi, I’m Cortland Burns! Over the last two years I’ve grown from building single-app solutions to delivering secure, cloud-ready systems across web and mobile. My focus is application development, full-stack design, and secure architecture—with hands-on practice improving auth flows, hardening APIs, integrating cloud databases (Firebase/AWS), and automating builds/tests.

JWT & API Security Firebase Auth/Firestore AWS (S3 • Lambda • API Gateway) Automation & CI MEAN Stack Flutter • SQLite

Engineering Highlights

Secure Software Growth

  • Hardened auth with JWT (expiry/refresh), role checks, and an Angular HttpInterceptor that injects tokens and handles 401/403 gracefully.
  • Backend hardening: Helmet, rate-limiting, strict CORS, input validation (Zod/Joi), bcrypt hashing, sanitized Mongo queries.
  • Secrets via dotenv + environment configs for dev/prod; minimal surface area on public endpoints.

Cloud & Data Integrations

  • Firebase: evaluating Auth for passwordless sign-in, Firestore for sync, and Hosting for rapid previews.
  • AWS: using S3 for static media/artifacts; exploring API Gateway + Lambda for serverless endpoints and webhooks.
  • Structured deployment configs to keep creds out of source and enable principle of least privilege.

Automation & APIs

  • API first: clean REST surface (auth, resources, admin), typed requests/responses, and testable contracts.
  • GitHub Actions to lint, test, and build MEAN/Flutter paths; optional deploy step to Pages/Firebase.
  • Postman/OpenAPI collections (WIP) for repeatable testing and onboarding.

Travlr Getaways (Full Stack MEAN)

A full-stack travel booking platform using MongoDB • Express • Angular • Node. Public site with Handlebars plus a secure Angular Admin SPA using JWT authentication and protected routes.

  • Source: cs465-fullstack repository
  • Demo Video: Travlr Getaways
  • Narrative (DOCX)

Security Improvements

  • JWT with expiration & refresh strategy; Angular JWT interceptor handles token attach & 401/403 flows.
  • bcrypt password hashing; strict CORS; Helmet headers; rate-limiting on auth & admin routes.
  • Request validation (e.g., Zod/Joi), sanitized Mongo queries, Mongoose schema validation & indexes.
  • Secrets via dotenv / environment; no secrets in source; principle of least privilege in DB users.

Cloud & Automation

  • Static/media to AWS S3; optional CloudFront for CDN; environment-based builds.
  • Serverless hooks via API Gateway + Lambda (notifications, audit logs, or webhooks).
  • GitHub Actions: lint → unit tests → build Angular → run API tests → (optional) deploy.

Automation & Security Setup (Quick Commands)

MEAN API Hardening

npm i helmet express-rate-limit cors bcrypt jsonwebtoken dotenv zod mongoose \
  mongo-sanitize
// example (Express)
import helmet from "helmet";
import rateLimit from "express-rate-limit";
import cors from "cors";

app.use(helmet());
app.use(cors({ origin: ["https://your-admin.app"], methods: ["GET","POST","PUT","DELETE"], credentials: true }));
app.use(rateLimit({ windowMs: 15 * 60 * 1000, max: 200 }));

Angular Admin (JWT Interceptor)

// attach token, redirect on 401/403, handle refresh if present
